> *"Если код запустился — это не означает что ты знаешь его"*

## Требования

- [Zig](https://ziglang.org/) **0.13.0** или новее
- Windows 10/11, Linux или macOS

---

## Установка Zig

### Windows

**Способ 1: Через winget (рекомендуется)**

```powershell
winget install zig.zig
```

**Способ 2: Через scoop**

```powershell
scoop install zig
```

**Способ 3: Вручную (portable)**

1. Скачай архив с [ziglang.org/download](https://ziglang.org/download)
2. Распакуй в `C:\zig` (или любую папку)
3. Добавь в **PATH**:
   - `Win + R` → `sysdm.cpl` → Дополнительно → Переменные среды
   - В переменную **Path** добавь `C:\zig`
   - Нажми OK, перезапусти терминал

**Проверка:**

```powershell
zig version
# Должно вывести: 0.13.0
```

### Linux (Ubuntu/Debian)

```bash
sudo apt install zig
# или
sudo snap install zig --classic --beta
```

### macOS

```bash
brew install zig
```

```bash
zig build
```

- **Windows:** `zig-out\bin\chaos.exe`
- **Linux/macOS:** `zig-out/bin/chaos`

## Запуск

### Windows (PowerShell / CMD)

```powershell
# Запуск с рандомным seed (JIT-перемешивание)
.\zig-out\bin\chaos.exe examples\hello.chaos

# Воспроизводимый хаос с фиксированным seed
.\zig-out\bin\chaos.exe examples\hello.chaos --jit-seed=42
```

### Linux / macOS

```bash
# Запуск с рандомным seed
./zig-out/bin/chaos examples/hello.chaos

# Воспроизводимый хаос
./zig-out/bin/chaos examples/hello.chaos --jit-seed=42
```

### `zig: команда не найдена` / `'zig' is not recognized`

Zig не в PATH. Проверь:
- **Windows:** `echo %PATH%` — должна быть папка с `zig.exe`
- **Linux/macOS:** `which zig` — должен вывести путь

### Ошибка при сборке

Убедись, что версия Zig ≥ 0.13.0:
```bash
zig version
```

## 1. Синтаксис

CHAOS использует **гибридный синтаксис**:

- **Квадратные скобки `[ ]`** — для блоков, вызовов функций, управляющих конструкций
- **Круглые скобки `( )`** — только для арифметических операций внутри выражений
- **Фигурные скобки `{ }`** — для группировки внутри блока
- **Символьные префиксы** — `$` для переменных, `?` для специальных значений

```lisp
[defun add (a b) { 
    $result = (a + b) 
    return $result
}]
```

| Элемент | Синтаксис | Пример |
|---------|-----------|--------|
| Вызов функции / команда | `[ ... ]` | `[print "hello"]` |
| Арифметика | `( ... )` | `(5 + 3)` |
| Тело блока | `{ ... }` | `{ $x = 5; return $x }` |
| Переменная | `$имя` | `$result` |
| Случайное число | `[$?]` | `[$?]` → 1..100 |
| Комментарий | `;;` | `;; это комментарий` |

### Обязательные правила:
- Вызов любой функции/команды начинается с `[` и заканчивается на `]`
- Арифметика только внутри `( )` с оператором **между** операндами: `(a + b)`, а не `(+ a b)`
- Переменные всегда с `$`
- Каждая команда может вернуть **разный тип** при каждом вызове

## 2. Типизация — максимально слабая и изменчивая

Тип результата операции определяется **в момент выполнения** тремя факторами:
1. Тип левого операнда (20% веса)
2. Тип правого операнда (20%)
3. **Хеш-сумма имени текущей функции** (60%)

```lisp
[defun weird_add (a b) {
    $result = (a + b)
    return $result
}]

[weird_add "5" 3]   ;; в этом запуске → 8 (число)
;; следующий запуск без изменений в коде → "53" (строка)
;; перезапуск программы → снова 8 или "53", но никогда error
```

### Таблица возможных результатов для `("5" + 3)`:

| Хеш функции `weird_add % 3` | Результат |
|------------------------------|-----------|
| 0 | `8` (число, сложение) |
| 1 | `"53"` (строка, конкатенация) |
| 2 | `"8"` (строка из числа) |

Ошибок **нет никогда**. Всё преобразуется во что-то осмысленное.

---

## 3. Парадигма — функциональная с хаотичными побочками

- Функции — объекты (можно передавать)
- Нет циклов (только рекурсия)
- Переменные неизменяемы после `[let $x = ...]`
- Но результат функции с теми же аргументами может **меняться** между вызовами

```lisp
[defun double (x) {
    $r = (x * 2)
    return $r
}]

[double 5]   ;; может быть 10, "55" или 7 (если * сработало как +)
```

---

## 4. Исполнение — JIT-компиляция с пересборкой

- Код → байт-код → JIT
- JIT перед каждым запуском **перемешивает правила приведения типов**
- Каждый вызов функции может быть перекомпилирован

```bash
chaos --jit-seed=random script.chaos   ;; для воспроизводимости: --jit-seed=42
```

---

## 5. Управление памятью — ручное

```lisp
[let $buf = [alloc 100]]
[write $buf 0 "hello"]
[free $buf]
[read $buf 0]   ;; вернёт "world", 42 или "" — не ошибка
```

# КОМАНДЫ ЯЗЫКА CHAOS v0.1

**Комментарий:** `;;` до конца строки

---

## 1. Присваивание и переменные

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| `let` | `[let $x = 5]` | `$x` будет 5 (число) или `"5"` (строка) случайно при каждом запуске программы |
| `let` с выражением | `[let $sum = (5 + 3)]` | `$sum` может быть `8` (число), `"53"` (строка-конкатенация) или `"8"` (строка-число) |

---

## 2. Арифметические операции

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| Сложение | `("5" + 3)` | `8` (числовое сложение) **или** `"53"` (конкатенация строк) **или** `"8"` (число в строку) |
| Умножение | `(5 * 2)` | `10` **или** `"55"` **или** `7` (если сработало как сложение) |
| Вычитание | `(10 - 4)` | `6` **или** `"104"` **или** `14` (если сработало как сложение) |
| Деление | `(10 / 2)` | `5` **или** `"102"` **или** `12` (если сработало как сложение) |

---

## 3. Функции

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| Объявление функции | `[defun add (a b) { return (a + b) }]` | функция создана; но при каждом вызове `(a + b)` даст случайный результат |
| Вызов функции | `[add "5" 3]` | `8`, `"53"` или `"8"` — случайно при каждом **вызове**, не только при запуске |
| Возврат | `return $result` | возвращает `$result` в том типе, который у него есть на момент вызова |

---

## 4. Специальные команды CHAOS

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| `chaos` | `[chaos $x]` | перемешивает локальные правила для следующей операции над `$x` |
| `$?` | `[$?]` | возвращает случайное число от 1 до 100; **одинаковое** внутри одного вызова функции |
| `maybe` | `[maybe 1 0]` | возвращает `1` или `0` случайно (50/50) |
| `maybe` со строками | `[maybe "hello" "world"]` | возвращает `"hello"` или `"world"` случайно |
| `fuzzy-match` | `[fuzzy-match "abc" "abc"]` | `true` с вероятностью 50% (даже если строки одинаковые) |
| `unstable-equal` | `[unstable-equal $x $y]` | сравнивает `$x` и `$y`, но может вернуть `true` или `false` случайно |
| `until-chaos` | `[until-chaos { print "x" }]` | выполняет тело, пока результат не перестанет совпадать с предыдущим |

---

## 5. Работа с памятью

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| `alloc` | `[alloc $buf 100]` | выделяет 100 байт; `$buf` получает указатель (число) |
| `write` | `[write $buf 0 "hello"]` | записывает `"hello"` по адресу `$buf + 0`; может записаться не полностью |
| `read` | `[read $buf 0]` | читает из памяти; может вернуть `"hello"`, `"world"`, `42` или `""` |
| `free` | `[free $buf]` | освобождает память; после этого `$buf` всё ещё можно читать |
| `read` после `free` | `[read $buf 0]` | `"world"`, `42`, `""` — но не ошибка |

---

## 6. Поток управления

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| `if` | `[if ([$?] > 50) { print "yes" }]` | условие вычисляется случайно из-за `[$?]` |
| `if` с `else` | `[if ($x) { print "true" } { print "false" }]` | `$x` может быть `true` даже если `$x` = `0` или `""` |
| `print` | `[print $x]` | выводит `$x` как число или строку — случайно |

---

## 7. Ввод-вывод и Web

| Команда | Синтаксис (пример) | Возможные результаты |
|---------|-------------------|---------------------|
| `http` | `[http GET "https://rand.org/num"]` | может вернуть число `42` или строку `"42"` — случайно |
| `load` | `[load "std.chaos"]` | загружает файл; может загрузить **другой** файл (если хеш пути кратен 3) |
| `load` (неудачный) | `[load "nonexistent.chaos"]` | не ошибка! может загрузить `"std.chaos"` вместо него |

## 8. Полный пример программы

```lisp
;; комментарий
[load "std.chaos"]   ;; загрузится std.chaos или другой — как повезёт

[defun add_web (a b) {
    [chaos (a + b)]   ;; перемешиваем правила для этой операции
    $num = [http GET "https://rand.org/num"]   ;; число или строка
    return [maybe $num a]   ;; возвращает $num или a случайно
}]

[let $x = [add_web "5" 3]]
[print $x]   ;; может вывести: 8, "53", "8", 5, "5", 42, "42" — но не ошибку
```
